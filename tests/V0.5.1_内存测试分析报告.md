## v0.5.1核心改动分析

### 旧逻辑（v0.5.0）：
```
输入 → 白名单 → 元边界 → HARD攻击 → 疆域分流（闸门）→ 未触发则放行
                                                    ↓
                                              COMBO/灰区检测（被跳过）
```

**问题**：疆域分流作为闸门，没触发疆域关键词的输入直接放行，COMBO和灰区根本不看 → 大量漏拦

### 新逻辑（v0.5.1）：
```
输入 → 白名单 → 元边界 → HARD攻击 → 疆域分流（仅标记）
                                                    ↓
                                          COMBO全量检测（不依赖疆域）
                                                    ↓
                                          灰区（仅疆域触发时检查）
```

**改进**：疆域分流不再作为闸门，COMBO对所有输入都检查

### 建议

1. 继续完善COMBO规则的触点覆盖
2. 考虑引入轻量级NLP模型辅助检测极难变体
